## Part 4: Implementation Notes

### Search Behavior

1. **Debounced Input**: Search triggers after 150ms of no typing
2. **Result Grouping**: Results grouped by type (Tasks, Projects, Users)
3. **Match Highlighting**: Matched text shown with special formatting
4. **Fuzzy Matching**: Supports fuzzy search for better discoverability
5. **Recent Searches**: Last 5 searches stored locally

### Command Palette Behavior

1. **Instant Filtering**: Commands filter as you type
2. **Fuzzy Matching**: `nt` matches "New Task"
3. **Shortcut Display**: Shows keyboard shortcuts for all commands
4. **Nested Menus**: Support for hierarchical command structure
5. **Recently Used**: Recent commands shown at top
6. **Context Awareness**: Commands change based on current view/selection

### Accessibility Considerations

1. **Never rely solely on color** - Always pair with icons/text
2. **Keyboard-first** - Full functionality without mouse
3. **Focus management** - Clear focus indicators
4. **Announcements** - Result count announced for screen readers
5. **Escape hatch** - `Esc` always closes/goes back

### Animation Guidelines (Level 4 Only)

1. **Search results**: Fade in from top, staggered 50ms
2. **Result selection**: Slide highlight, 100ms ease-out
3. **Preview panel**: Slide in from right, 150ms ease-out
4. **Command execution**: Brief flash confirmation
5. **No results**: Gentle fade in for empty state

interface Command {

  label: string;
  description?: string;

    const base = path.length > 0
      ? path[path.length - 1].children || []
      : commands;

    if (!query) return base;

      cmd.label.toLowerCase().includes(query.toLowerCase())
    );
  }, [commands, path, query]);

  // Group by category
  const groupedCommands = useMemo(() => {
    const groups: Record<string, Command[]> = {};
    for (const cmd of currentCommands) {
      const group = cmd.group || 'Other';
      if (!groups[group]) groups[group] = [];
      groups[group].push(cmd);
    }
    return groups;
  }, [currentCommands]);

  const flatCommands = useMemo(() => {
    return Object.values(groupedCommands).flat();
  }, [groupedCommands]);

  useInput((input, key) => {
    if (!isFocused) return;

    if (key.escape) {
      if (path.length > 0) {
        setPath((p) => p.slice(0, -1));
        setQuery('');
      } else {

      }
      return;

        setQuery('');
        setSelectedIndex(0);
      } else {
        cmd.action();
        onClose();

      setQuery('');
      setSelectedIndex(0);

    if ((key.leftArrow || key.backspace) && path.length > 0 && !query) {
      setPath((p) => p.slice(0, -1));
      setSelectedIndex(0);
    }
  });

  const highlightMatch = (text: string, query: string) => {
    if (!query) return text;
    const regex = new RegExp(`(${query})`, 'gi');
    const parts = text.split(regex);
    return parts.map((part, i) =>
      regex.test(part) ? (
        <Text key={i} color="cyan" bold>
          {part}
        </Text>
      ) : (
        part
      )
    );
  };

  return (
    <Box
      flexDirection="column"
      borderStyle="round"
      borderColor="#3F3F46"
      paddingX={1}
    >
      {/* Header with breadcrumb */}
      <Box marginBottom={1}>
        <Text color="#3B82F6">âŒ˜</Text>
        {path.length > 0 ? (
          <>
            {path.map((p, i) => (
              <React.Fragment key={p.id}>
                <Text color="#71717A"> â€º </Text>
                <Text>{p.label}</Text>
              </React.Fragment>
            ))}
          </>
        ) : (
          <Text bold> Command Palette</Text>
        )}
        {path.length > 0 && (
          <Box marginLeft={2}>
            <Text color="#3B82F6">â€¹ Back</Text>
          </Box>
        )}
      </Box>

      {/* Search Input */}
      <Box
        borderStyle="round"
        borderColor={isFocused ? '#3B82F6' : '#3F3F46'}
        paddingX={1}
        marginBottom={1}
      >
        <TextInput
          value={query}

## Summary

The Search and Command Palette patterns provide:

1. **Progressive Enhancement** - Same information at all 4 rendering levels
2. **Consistent Semantics** - Symbols and colors mean the same thing everywhere
3. **Keyboard-First** - Full functionality without mouse
4. **Agent-Friendly** - Parseable output at every level
5. **Accessible** - Never relies solely on color for meaning

Key design decisions:

- **Modal overlay** for focused interaction
- **Grouped results** for scanability
- **Match highlighting** for search relevance
- **Keyboard hints** always visible
- **Preview panels** for context without navigation
- **Breadcrumb navigation** for nested commands
- **Semantic colors** for action types (create, delete, navigate)

These patterns integrate with the broader SaaSkit design system and can be composed with other views (List, Detail, Form) for complete application interfaces.
