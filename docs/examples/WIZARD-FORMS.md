# Multi-Step Wizard Forms

## Reference Implementation: "Create Project" Wizard

This document provides production-quality reference mockups for multi-step wizard forms across all four rendering levels of the SaaSkit design system.

### Wizard Structure

```
Step 1: Basic Info    - Project name, description
Step 2: Team          - Owner selection, member multi-select
Step 3: Settings      - Visibility toggle, notification checkboxes
Step 4: Review        - Summary of all entries, final confirmation
```

---

## Level 1: Plain Text (Alphanumeric Only)

Plain text renders with letters, numbers, and basic punctuation only. No box-drawing characters. Hierarchy is conveyed through whitespace and CAPS.

## Usage Guidelines

1. **Progressive Disclosure**: Show only relevant fields for each step. Don't overwhelm users with all options at once.

2. **Validation Timing**: Validate fields on blur for simple fields, on submit for complex validation.

3. **Step Navigation**: Allow users to go back and edit previous steps. Preserve entered data.

4. **Completed Steps Summary**: Show a condensed view of completed steps for context.

5. **Final Review**: Always provide a comprehensive review step before final submission.

6. **Loading States**: Show clear feedback during submission with progress indication.

7. **Error Recovery**: Make it easy to identify and fix errors without losing progress.

8. **Keyboard First**: Ensure complete keyboard navigation. Users should never need a mouse.

9. **Graceful Degradation**: Design for Level 1 first, then enhance. Never require color or special characters for comprehension.

10. **Agent Compatibility**: Provide JSON output option for automation and AI agent integration.
